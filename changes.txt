DECISIONS LANDED
================

Naming
------
The two Lumen engines are now branded:
  - Lumen Intraday: thirty minute scans, multi instrument, Claude
    Sonnet 4.6 decision model. (Was "Autotrader".)
  - Lumen Scalper:  sixty second scans, user selected instruments,
    Claude Haiku 4.5 decision model. (Unchanged.)

Intraday is the precise financial term for entries opened and closed
within a single trading day. Scalper is the standard industry term
for sub five minute holds. The sidebar nav was renamed to "Lumen"
since the tab actually contains both engines.

Worker retirement
-----------------
The Cloudflare Worker at worker/index.js was duplicated by Vercel
routes that also have Upstash Redis caching the Worker lacks. All
Lumen calls now hit Vercel directly. The Worker is left in place
but no client code calls it. Safe to delete after a stable deploy
or two.

Speed sensitive paths bypass both gateways:
  - Crypto prices: Binance WebSocket direct from browser
  - Forex and metals prices: Deriv WebSocket direct from browser
  - Crypto candles: Binance REST direct
  - Forex/metals/indices/commodities prices: cached
    /api/prices route (Vercel + Redis)
  - Non crypto candles: cached /api/candles route (Vercel + Redis)

Models
------
  - Big picture market intelligence (Markets tab via /api/scan
    default): claude-opus-4-7 (upgraded from 4.6)
  - Lumen Intraday: claude-sonnet-4-6 via /api/scan body model param
    (Opus would be overkill at thirty minute cadence)
  - Lumen Scalper: claude-haiku-4-5-20251001 via /api/behaviour
  - Sentiment poller: routed through /api/sentiment which fans out
    Grok 3 → Claude Haiku 4.5 → Gemini 2.5 Flash server side
  - Server side fallback to Gemini 2.5 Flash on every Lumen route

Confidence thresholds and risk
------------------------------
  - Intraday: seventy five and above. Picks the highest cleared
    signal across the day's three instruments per cycle. Maximum
    three open positions.
  - Scalper weekday: sixty five and above
  - Scalper weekend: seventy five and above (thinner liquidity)

Daily budgets (UTC reset)
-------------------------
  - Intraday: 200 calls
  - Scalper:  1500 calls
  - Sentiment: 300 calls
  - Selection: 30 calls

WHAT WAS BROKEN BEFORE THIS SESSION
====================================

Five distinct bugs all reinforcing each other:

1. URL prefix bug. WORKER_URL was '/api'. Lumen built fetches as
   WORKER_URL + '/api/gemini' giving '/api/api/gemini' (404). Every
   Lumen call hit a Vercel 404. Other modules used WORKER_URL +
   '/scan' which resolved to '/api/scan' correctly, so markets and
   academy worked. Only Lumen broke.

2. Fictional model strings. Lumen referenced gemini-3.1-flash-lite-
   preview, gemini-3.1-pro-preview, grok-3-fast, grok-3-reasoning.
   None of these exist. Every Gemini and Grok call would have
   returned 400 even if the URL was correct.

3. Fallback chains depended on broken models. Sentiment, dataprep,
   decision, and scalp chains all started with a Gemini 3.1 step
   that failed, then attempted Grok 3 fast (does not exist), then
   Claude via the wrong URL.

4. Hardcoded fake indicators. atRunScan passed
   {price, rsi: 62, ema20: 59000, ema50: 58500} to the LLM. Every
   scan asked Sonnet to make a $5,000 lot decision on constant
   numbers.

5. Schema mismatch. DECISION_SYSTEM asked for {verdict, ...} while
   atRunScan checked result.verdict. SCALP_SYSTEM asked for
   {action, ...} while scalpRunScan checked signal.action. Without
   defensive parsing, the autotrader silently swallowed every
   exception and the engine ran but never traded.
