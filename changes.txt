REFERENCE: LUMEN INTELLIGENCE NAMESPACE
---------------------------------------
Every Batch 1 to 4 client side intelligence module exposes a function
under the global window.LumenIntel object:

  await LumenIntel.yields()                -> real yield analysis
  await LumenIntel.cot('gold')             -> COT analysis (live)
  await LumenIntel.wyckoff('XAUUSD')       -> phase, sub_phase, narrative
  await LumenIntel.regime('XAUUSD')        -> trend + volatility chars
  await LumenIntel.correlations()          -> matrix + breakdowns
  await LumenIntel.liquidity('XAUUSD')     -> 3 above + 3 below
  await LumenIntel.calendar('XAUUSD')      -> pre/post event flags
  LumenIntel.session()                     -> kill zone multiplier
  LumenIntel.portfolio()                   -> exposure + clusters
  LumenIntel.correlations.lookup(a, b)     -> cached corr value
  LumenIntel.cot.legacyCacheRow('gold')    -> markets.js shape
  LumenIntel.<any>.cached(...)             -> last cached payload
  LumenIntel.<any>.clear(...)              -> wipe local cache

All intel modules degrade gracefully to a neutral or null payload on
any failure so the master synthesis prompt never blocks waiting for
an unreachable upstream.

REFERENCE: PRICE PERSISTENCE
----------------------------
prices.js writes a snapshot of livePriceCache, priceSourceCache and
priceTimestampCache to localStorage under wm_price_cache on every
update (debounced to once every three seconds). On script load it
hydrates all three caches from this snapshot. Consequence: a Saturday
morning cold load shows Friday's close for every previously seen
forex, metals, indices and commodities pair, with a small CLOSED tag
in the markets table and a dim C suffix in the ticker. A 60 second
freshness threshold is applied via priceIsStale(pairId).

REFERENCE: SCALPER AUTO CLOSE
-----------------------------
scalpMonitorPositions runs at the top of each scalpRunScan tick.
For every open scalp position it reads the live price, checks for
SL or TP cross (direction aware), computes the pnl and the R
multiple if a stop distance is defined, and pushes a closed record
into scalpHistory with closePrice, pnl, r_multiple, closedAt, and
exit_reason fields plus the original journal_context block. The
pattern miner now sees scalp closes alongside Intraday closes.
scalpHistory is capped at 200 entries.
