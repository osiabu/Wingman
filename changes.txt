Architectural decisions landed at the start of this run:
  - MT5 demo execution skipped. Wingman stays simulated only. Conflicts
    with the Terms of Service in index.html which states all trades are
    simulated and no real money is at risk at any time.
  - GVZ unavailable as a free public feed. Substitute approved: VIX
    (already wired in markets.js fetchFearGreed) plus an ATR percentile
    compression detector built on local candles inside the regime
    classifier.
  - TimescaleDB / PostgreSQL skipped. Browser only architecture stays.
    Substitutes: localStorage TTL caches client side, Upstash Redis
    server side for shared module outputs.
  - Daily LLM budgets stay at current caps (intraday 200, scalper 1500,
    sentiment 300, select 30). Aggressive cache TTLs (Wyckoff 4h keyed
    by H4 bar timestamp, COT 7d, real yields 24h, correlations 4h,
    liquidity 1h, calendar 15m) prevent the master synthesis prompt
    from multiplying per cycle LLM calls.

NAMESPACE
---------
Every Batch 1 to 4 client side intelligence module exposes a function
under the global window.LumenIntel object:

  await LumenIntel.yields()                -> real yield analysis
  await LumenIntel.cot('gold')             -> COT analysis (live)
  await LumenIntel.wyckoff('XAUUSD')       -> phase, sub_phase, narrative
  await LumenIntel.regime('XAUUSD')        -> trend + volatility chars
  await LumenIntel.correlations()          -> matrix + breakdowns
  await LumenIntel.liquidity('XAUUSD')     -> 3 above + 3 below
  await LumenIntel.calendar('XAUUSD')      -> pre/post event flags
  LumenIntel.session()                     -> kill zone multiplier
  LumenIntel.portfolio()                   -> exposure + clusters
  LumenIntel.correlations.lookup(a, b)     -> cached corr value
  LumenIntel.cot.legacyCacheRow('gold')    -> markets.js shape
  LumenIntel.<any>.cached(...)             -> last cached payload
  LumenIntel.<any>.clear(...)              -> wipe local cache

All intel modules degrade gracefully to a neutral or null payload on
any failure so the master synthesis prompt never blocks waiting for an
unreachable upstream.

DECISION FLOW (Lumen Intraday, every 30 minutes)
------------------------------------------------
1. intradaySelectInstruments() picks three instruments via Sonnet
   (cached per UTC day).
2. For each instrument:
   a. lumenIndicators(inst) computes RSI, EMA20, EMA50.
   b. assembleIntradayIntel(inst, indicators) gathers the full intel
      payload. This may trigger the Wyckoff LLM call (cached 4h) and
      candle pulls; everything else is cached and free.
   c. Hard skip if calendar.pre_event_caution is true.
   d. intradayDecide(inst, intel) sends the full payload to Sonnet 4.6.
   e. Hard skip on SKIP action.
   f. lumenPortfolioGate rejects duplicate same direction trades.
   g. applySessionMultiplier scales raw confidence to effective.
   h. Track the highest effective confidence across the three.
3. If the best effective confidence is at or above 75, execute. The
   position record carries the full intel snapshot for pattern mining.

DECISION FLOW (Lumen Scalper, every 60 seconds)
-----------------------------------------------
1. Filter to crypto on weekends. Confidence threshold raised to 75 on
   weekends, 65 on weekdays.
2. For each user selected instrument:
   a. Read live price from livePriceCache.
   b. lumenIndicators(inst) for crypto only.
   c. assembleScalperIntel reads cached Wyckoff, regime, liquidity,
      calendar, session. NEVER triggers an LLM call.
   d. Hard skip if pre_event_caution is true.
   e. scalpDecide sends the slim payload to Haiku 4.5.
   f. lumenPortfolioGate rejects duplicates across all engines.
   g. applySessionMultiplier scales raw to effective.
3. Execute if effective is at or above the threshold.
